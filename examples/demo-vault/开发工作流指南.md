# 开发工作流指南

## Claude 与 Obsidian 的协作机制

这个 vault 是 Claude Code 的"上下文缓存"：Claude 在每次开工前先读 vault 获取项目状态，避免从零扫代码。

### 查阅优先级

1. **先读 Obsidian** — 架构、当前焦点、跨仓库依赖在文档里
2. **需要时读代码** — 文档不够时再去看具体实现
3. **发现不一致时更新文档** — 代码和文档冲突时，主动修正文档
4. **不要盲扫** — 不要为了"理解架构"遍历整个代码目录

### 三种文档的角色

| 文档 | 读的时机 | 写的时机 |
|------|---------|---------|
| `概览.md` | 第一次接触项目 / 解释架构时 | 架构有重大变化 |
| `activeContext.md` | **每次开工前必读** | 切换工作焦点时 |
| `progress.md` | 周回顾、汇报进度时 | 完成里程碑时 |

### 命令对应

| 场景 | 命令 |
|------|------|
| 改完代码同步到文档 | `/sync-docs` |
| 提交代码 + 同步文档 | `/commit` |
| 拉代码 + 更新文档 | `/pull` |
| 切换焦点，更新当前在做啥 | `/update-active` |
| 周回顾 | `/weekly-digest` |

### 跨仓库联动

修改 [[项目/example-shared-proto/概览]] 这种共享库后，必须在所有下游项目的 `activeContext.md` 加一条「待联动」，避免遗漏。
