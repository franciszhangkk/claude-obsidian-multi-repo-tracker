# example-api-server 概览

> 项目入口文档。第一次接触项目或解释架构时读这里。

## 一句话定位

订单系统后端 API 服务，处理订单创建、查询、状态流转。

## 代码位置

- **仓库路径**：`~/code/example-api-server`
- **主语言**：Go 1.23
- **主要依赖**：gRPC、PostgreSQL、Redis、[[项目/example-shared-proto/概览]]

## 架构速览

分三层：HTTP 入口（gin）→ 业务层（service）→ 数据层（repository）。订单状态机在 service 层，事件通过 Redis Pub/Sub 通知前端。proto 接口定义来自 example-shared-proto。

## 关键模块

| 模块 | 路径 | 职责 |
|------|------|------|
| HTTP API | `internal/api/` | 路由 + 请求校验 |
| 订单服务 | `internal/service/order/` | 状态机、业务规则 |
| Repository | `internal/repo/` | 数据库 + 缓存 |
| gRPC Client | `internal/grpc/` | 调用其他服务 |
