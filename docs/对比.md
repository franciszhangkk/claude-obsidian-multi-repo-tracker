## 详细对比

### vs Cline Memory Bank

[Cline Memory Bank](https://docs.cline.bot/features/memory-bank) 是六文件固定结构（projectbrief / productContext / activeContext / systemPatterns / techContext / progress）。

**借鉴**：activeContext + progress 的设计直接拿来用。
**差异**：
- 我们是"vault 多项目"模式，Cline 是"单 repo 单 memory-bank"
- 我们额外管"跨仓库依赖"
- 我们绑定 git 同步（Cline 不管这块）
- 我们文件分类更细（功能/产品/参考资料/测试调试）

**适合 Cline 的人**：单项目，结构简单，英文环境。
**适合本项目的人**：多个相关代码仓库，需要统一索引。

### vs obsidian-second-brain

[obsidian-second-brain](https://github.com/eugeniughelbur/obsidian-second-brain) 是通用的 second brain，支持自主写入、思考工具、知识摄入、定时 agent。

**差异**：定位完全不同
- 它管"个人知识沉淀"（笔记、想法、外部资料）
- 我们管"开发项目跟踪"（架构、进度、跨仓库）
- 两者**可以并存**在同一个 vault 里，文件夹分开即可

### vs claude-obsidian (Karpathy LLM Wiki 实现)

[claude-obsidian](https://github.com/AgriciDaniel/claude-obsidian) 实现了 Karpathy 的 LLM Wiki 模式：raw sources → wiki → schema 三层架构，重点是"LLM 编译外部资料"。

**差异**：
- LLM Wiki 是"知识编译器"（你扔文章，LLM 整理）
- 我们是"项目状态板"（你写代码，文档跟着更新）
- 两套思路可以**互补**：用 LLM Wiki 沉淀外部知识，用本项目跟踪开发工作

### vs claude-code-memory-setup

[claude-code-memory-setup](https://github.com/lucasrosati/claude-code-memory-setup) 用 Obsidian + Graphify 做知识图谱，号称省 71.5x token。

**差异**：
- 它重图谱（向量 + 关系）
- 我们重结构化文档（纯 markdown 直读，无依赖）
- 它复杂度高（要装 Graphify、配置图谱），我们安装即用

## 该选哪个

| 你的需求 | 推荐 |
|---------|------|
| 单项目，想快速试试 memory bank 模式 | Cline Memory Bank |
| 多项目，跨仓库依赖，需要"项目跟踪" | **本项目** |
| 重个人知识沉淀，轻代码跟踪 | obsidian-second-brain |
| 想把外部文章/论文用 LLM 编成 wiki | claude-obsidian (Karpathy) |
| 重知识图谱，能接受复杂配置 | claude-code-memory-setup |

不互斥——本项目和 second-brain / Karpathy LLM Wiki 完全可以**同时**用在一个 vault 里。
